## 1. 项目概述

### 1.1 项目背景

理光中国后台管理系统是为理光中国公司打造的企业级综合管理平台，旨在实现用户管理、报备管理、权限控制、内容发布、商品管理、订单处理等核心业务的数字化和自动化。

### 1.2 项目目标

- **提升运营效率**: 通过自动化流程减少人工操作时间
- **规范业务流程**: 建立标准化的报备审批和订单处理流程
- **数据可视化**: 提供实时数据统计和报表分析
- **权限精细化**: 实现基于角色的访问控制
- **客户体验优化**: 提供便捷的内容管理和订单跟踪

### 1.3 核心价值

- **智能报备系统**: 根据客户类型和角色自动路由审批流程
- **库存协同管理**: 打通经销商库存，实现实时库存确认
- **内容一键导入**: 支持从外部URL自动抓取并导入内容
- **全流程追踪**: 从报备到订单完成的全生命周期管理

### 4.2 报备管理模块

#### 功能概述
实现智能化的客户报备和审批流程，根据客户类型和提交人角色自动分配审批路由。

#### 4.2.1 客户类型

| 客户类型 | 说明 | 审批流程 |
|---------|------|---------|
| 普通客户 | 未在系统中登记的客户 | 标准审批流程 |
| RGS客户 | 区域重点客户 | 区域经理审批 |
| NAM客户 | 全国重点客户 | 专项审批 |
| 央国企大客户 | 央企/国企客户 | 大客户协调专审 |
| 框架客户 | 签署框架合同客户 | 区域渠道支援审批 |

#### 4.2.2 用户角色

| 角色 | 权限 | 审批流程 |
|-----|------|---------|
| 一级代理商 | 直接报备权限 | 区域渠道支援审批 |
| 二级代理商 | 需初审的报备权限 | 一级代理商初审 → 区域渠道支援终审 |
| 直销经理 | 全类型客户报备 | 根据客户类型智能路由 |
| 区域渠道支援 | 审批权限 | 终审角色 |
| 市场部担当 | 审批权限 | 直销普通客户审批 |
| 直销报备协调 | 审批权限 | 直销内部冲突处理 |
| 区域经理 | 审批权限 | RGS/NAM客户初审 |
| 直销副总经理 | 最终决策权 | RGS/NAM客户终审 |
| 大客户报备协调 | 专审权限 | 央国企/框架客户审批 |

#### 4.2.3 业务流程

**流程1：一级代理商报备流程**
```
1. 输入"公司名+二级部门名称"检索客户
2. 系统执行全局静默查重
3. 返回客户类别（普通客户）
4. 选择品类/区域，提交报备材料
5. 系统自动流转至区域渠道支援
6. 审批结果：通过/拒绝/要求补充
7. 更新客户归属数据库
8. 异步发送通知
```

**流程2：二级代理商报备流程**
```
1. 提交报备申请
2. 系统通知上级一级代理商
3. 一级代理商初审：
   - 同意 → 流转至区域渠道支援
   - 拒绝 → 流程终止
4. 区域渠道支援终审
5. 通知二级代理商结果
```

**流程3：直销经理报备流程**
```
根据客户类型智能路由：

【普通客户】
- 正常报备 → 市场部担当（姚俊）
- 跨渠道申诉 → 市场部担当（姚俊）
- 直销内部冲突 → 直销报备协调（黄迪华/周明）

【RGS/NAM客户】
- 报备 → 区域经理初审
  - 同意 → 报备成功
  - 拒绝 → 上诉至直销副总经理（戴总）终审

【央国企大客户】
- 自动流转至大客户报备协调（王凡/连紫昭）
```

**流程4：央国企/框架客户专审**
```
1. 系统自动识别特殊客户
2. 【央国企】→ 大客户报备协调审批
3. 【框架客户】→ 区域渠道支援审批
4. 优先处理通道
5. 通知报备结果
```

**流程5：数据库自动化管理**
```
1. 大客户协调人维护客户数据库
2. 添加框架合同到期时间
3. 定时任务每日检查：
   - 到期超过宽限期 → 自动设为"失效"
   - 合作成功标记 → 释放至"可报备库"
4. 状态自动流转
```

#### 4.2.4 核心功能

### 4.6 订单管理模块

#### 功能概述
管理客户订单的完整生命周期，协调经销商库存和物流配送。

#### 4.6.1 订单状态流转

```
待确认库存 → 无库存取消
           ↓
        有合同？
           ↓
    ┌──────┴──────┐
    │             │
    ↓             ↓
待客户签署合同   待付款
    ↓             │
待理光签署        │
    ↓             │
    └──────┬──────┘
           ↓
        待付款 → 已付款 → 已发货 → 已完成
           ↓
        已取消
```

#### 4.6.2 订单处理流程

**步骤1：库存确认**
```
1. 客户下单
2. 系统根据客户地址自动匹配经销商
3. 显示经销商联系信息（姓名/电话）
4. 运营人员电话联系经销商询问库存
5. 在后台填写库存确认信息：
   - 选择：有库存/无库存
   - 填写备注（库存数量、预计发货时间等）
6. 无库存 → 订单变为"无库存取消"，通知客户
7. 有库存 → 系统判断是否需要合同：
   - 需要合同 → 订单变为"待客户签署合同"，通知客户
   - 不需要合同 → 订单变为"待付款"，通知客户
```

**步骤1.1：合同签署（有合同的订单）**
```
1. 订单变为"待客户签署合同"状态
2. 客户在前台查看合同并签署
3. 客户签署后，订单变为"待理光签署"状态
4. 理光相关人员审核并签署合同
5. 理光签署后，订单变为"待付款"状态
6. 通知客户付款
```

**步骤2：填写物流**
```
1. 客户付款后，订单变为"已付款"
2. 经销商发货后，获取物流信息
3. 运营人员在后台填写：
   - 物流公司（下拉选择）
   - 物流单号
4. 提交后订单变为"已发货"
5. 客户在前台自动收到物流通知
6. 可查看物流追踪信息
```

**步骤3：确认收货**
```
1. 客户收到货物
2. 在前台点击"确认收货"
3. 后台订单状态变为"已完成"
4. 记录收货时间
5. 进入报表统计
```

└───────────────────────────────────────────────────┘

### 8.2 订单流转协同

```
【前台】客户下单
  ↓
【数据库】创建订单（状态：待确认库存）
  ↓ 自动匹配经销商
【后台】运营人员查看新订单
  ↓ 电话联系经销商
【后台】确认库存结果
  ↓ 有货/无货
【数据库】更新订单状态
  ↓ 有货→待付款 | 无货→已取消
【前台】用户收到通知
  ↓ 有货则去付款
【前台】用户付款
  ↓
【数据库】更新订单状态（已付款）
  ↓ 经销商发货
【后台】填写物流信息
  ↓
【数据库】更新订单状态（已发货）
  ↓
【前台】用户查看物流
  ↓ 收到货物
【前台】确认收货
  ↓
【数据库】更新订单状态（已完成）
  ↓
【后台】生成报表统计
```

### 8.3 报备流转协同

```
【前台】代理商提交报备
  ↓
【数据库】创建报备记录（待审批）
  ↓
【后台】系统执行客户查重
  ↓
【后台】智能审批路由分配
  ↓
【数据库】更新报备状态（审批中）
  ↓
【前台】代理商查看审批进度
  ↓
【后台】审批人审阅处理
  ↓
【数据库】更新审批结果
  ↓
【前台】代理商收到审批通知
  ↓
【后台】更新客户归属数据库
```

### 8.4 实时通信机制

#### 8.4.1 WebSocket实时推送

**推送场景**:
- 订单状态变更实时推送
- 报备审批结果实时通知
- 系统消息即时送达
- 在线客服实时对话

**技术实现**:
```
前台建立WebSocket连接
  ↓
后台订阅用户消息队列
  ↓
业务事件触发（订单/报备状态变更）
  ↓
后台推送消息到前台
  ↓
前台接收并展示通知
```

#### 8.4.2 轮询机制（备选）
```
前台定时请求接口（30秒间隔）
  ↓
后台返回未读消息
  ↓
前台展示消息提示
  ↓
用户点击查看详情
```

### 8.5 数据同步策略

#### 8.5.1 商品数据同步
```
【后台】管理员发布/更新商品
  ↓
【数据库】商品表更新
  ↓
【缓存】Redis缓存更新

【前台】自动刷新商品信息

#### 8.5.2 价格策略同步

【前台】根据用户类型显示价格

  - 普通用户：标准价

  - VIP用户：专属价（最优惠）
```

## 16. 常见问题（FAQ）

### 16.1 报备相关

**Q: 如何判断客户类型？**
A: 系统通过"公司名+部门名"进行全局查重，自动判定客户类型。

**Q: 二级代理商为什么需要两次审批？**
A: 二级代理商需要一级代理商背书，先由一级代理商初审，通过后再由区域渠道支援终审。

**Q: 如何处理报备冲突？**
A: 选择"冲突申诉"类型，提供完整证据材料，系统会流转至对应审批人处理。

### 16.2 订单相关

**Q: 为什么看不到商品库存？**
A: 实际库存在各地经销商处，需要运营人员电话联系经销商确认。

**Q: 如何修改物流信息？**
A: 在订单详情页可以重新填写物流信息，会覆盖之前的记录。

**Q: 客户如何查看物流信息？**
A: 订单发货后，客户在前台订单详情页可以看到物流公司和单号。

### 16.3 内容相关

**Q: URL导入功能支持哪些网站？**
A: 目前为演示模式，实际需要后端API支持。支持大部分新闻网站和博客。

**Q: 导入的内容可以修改吗？**
A: 可以，导入后自动填充到表单，用户可以编辑调整后再发布。
