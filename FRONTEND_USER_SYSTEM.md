## 1. 系统概述

### 1.1 前台系统定位

理光中国前台用户系统是面向终端客户、代理商、经销商的前端应用平台，与后台管理系统形成完整的业务闭环。

**面向对象**:
- 终端企业客户
- 一级/二级代理商
- 经销商合作伙伴

**核心功能**:
- 商品浏览与在线下单
- 客户报备申请
- 订单全流程跟踪
- 个人信息管理

**设计理念**:
- ✅ 简洁直观 - 降低使用门槛
- ✅ 移动优先 - 响应式设计
- ✅ 流程透明 - 全程可视化
- ✅ 实时反馈 - 即时通知

### 1.2 系统特点

- ✅ **响应式设计**: 完美适配PC/平板/手机
- ✅ **即时注册**: 用户注册即可使用，无需等待审核
- ✅ **实时同步**: 与后台数据实时同步
- ✅ **消息推送**: 订单/报备状态及时送达
- ✅ **全程追踪**: 订单报备全生命周期可视化

## 2. 核心功能模块

### 2.1 用户注册与登录

#### 2.1.1 注册流程

```
步骤1：访问前台网站
  ↓
步骤2：点击"立即注册"
  ↓
步骤3：填写注册信息
  - 手机号（11位，作为登录账号） *
  - 密码（8-20位，需含字母和数字） *
  - 短信验证码（6位） *
  - 公司名称 *
  - 真实姓名 *
  - 所在地区（省市区三级联动） *
  - 行业类型（下拉选择） *
  ↓
步骤4：阅读并勾选《用户协议》和《隐私政策》
  ↓
步骤5：点击"立即注册"
  ↓
步骤6：注册成功，自动登录并跳转到首页
```

**注册页面设计要点**:
- 左侧：品牌形象展示/产品优势/成功案例
- 右侧：注册表单（卡片式设计）
- 表单验证：实时验证（手机号格式/密码强度）
- 短信验证码：60秒倒计时重发
- 错误提示：友好的错误信息

#### 2.1.2 登录方式

**方式一：手机号+密码登录**
```
1. 输入手机号
2. 输入密码
3. 勾选"记住我"（可选，保持7天登录状态）
4. 点击"登录"
```

**方式二：手机号+验证码登录**
```
1. 输入手机号
2. 点击"获取验证码"
3. 输入收到的6位验证码
4. 点击"登录"
```

**其他功能**:
- 🔑 忘记密码：通过短信验证码重置
- 🔒 账号安全：登录异常时要求验证
- 📱 第三方登录：微信扫码登录（可选）

#### 2.3.3 支付页面

**在线支付**:
```
1. 提交订单后跳转到支付页面
2. 显示订单号和应付金额
3. 显示支付方式选择：
   - 支付宝扫码支付
   - 微信扫码支付
   - 企业网银支付
4. 用户完成支付
5. 支付成功，跳转到订单详情页
6. 系统发送付款成功通知
```

**线下转账**:
```
1. 选择"线下转账"
2. 提交订单成功
3. 显示公司银行账户信息
4. 用户线下转账
5. 上传付款凭证
6. 等待财务确认
7. 财务确认后订单变为"已付款"
```

### 6.2 订单全流程协同

```
【前台】客户浏览商品并下单
  ↓
【数据库】创建订单记录
  - 状态：pending_stock（待确认库存）
  - 自动匹配经销商（根据客户城市）
  ↓
【后台】运营人员收到新订单提醒
  ↓
【后台】查看订单详情和经销商联系方式
  ↓
【后台】电话联系经销商确认库存
  ↓
【后台】填写库存确认结果
  - 有库存：填写备注（数量/发货时间）
  - 无库存：填写备注（原因说明）
  ↓
【数据库】更新订单状态
  - 有库存 → pending_payment（待付款）
  - 无库存 → out_of_stock（无库存取消）
  ↓
【前台】用户收到通知
  - 有库存：提示去付款
  - 无库存：订单已取消
  ↓
【前台】用户在线支付或线下转账
  ↓
【数据库】更新订单状态
  - 状态：paid（已付款）
  - 记录支付时间
  ↓
【后台】查看已付款订单
  ↓
【经销商】发货并提供物流信息
  ↓
【后台】运营人员填写物流信息
  - 物流公司
  - 物流单号
  ↓
【数据库】更新订单状态
  - 状态：shipped（已发货）
  - 记录发货时间
  ↓
【前台】用户收到发货通知
  - 查看物流公司和单号
  - 点击查看物流追踪
  ↓
【前台】用户收到货物后确认收货
  ↓
【数据库】更新订单状态
  - 状态：completed（已完成）
  - 记录收货时间
  ↓
【后台】订单完成，进入报表统计
```

### 6.3 报备全流程协同

```
【前台】代理商填写报备申请
  ↓
【数据库】创建报备记录
  - 状态：pending（待审批）
  - 生成报备编号
  ↓
【后台】系统执行全局客户查重
  ↓
【后台】智能审批路由分配审批人
  - 根据客户类型
  - 根据提交人角色
  - 根据报备区域
  ↓
【数据库】更新报备状态
  - 状态：approving（审批中）
  - 记录当前审批人
  ↓
【前台】代理商查看审批进度
  - 显示当前审批人
  - 显示预计审批时间
  ↓
【后台】审批人收到待审批提醒
  ↓
【后台】审批人查看报备详情
  - 客户信息
  - 报备说明
  - 附件材料
  ↓
【后台】审批人做出决定
  - 通过：填写审批意见
  - 拒绝：填写拒绝原因
  - 补充：说明需要补充的材料
  ↓
【数据库】更新审批结果
  - 状态：approved/rejected/need_supplement
  - 记录审批意见
  - 记录审批时间
  ↓
【前台】代理商收到审批通知
  - 通过：显示审批意见
  - 拒绝：显示拒绝原因，可申诉
  - 补充：显示需补充内容，可重新提交
  ↓
【后台】更新客户归属数据库
  - 报备通过后锁定客户
  - 分配给代理商
  - 设置保护期
```

#### 6.4.1 WebSocket实时推送

```javascript
// 前台建立WebSocket连接
const ws = new WebSocket('wss://api.ricoh.com/ws');

ws.onopen = () => {
  // 发送认证信息
  ws.send(JSON.stringify({
    type: 'auth',
    token: localStorage.getItem('token')
  }));
};

ws.onmessage = (event) => {
  const message = JSON.parse(event.data);
  
  switch(message.type) {
    case 'order_update':
      // 订单状态更新
      showNotification('订单更新', message.content);
      updateOrderStatus(message.order_id);
      break;
      
    case 'report_approval':
      // 报备审批结果
      showNotification('报备审批', message.content);
      updateReportStatus(message.report_id);
      break;
      
    case 'system_message':
      // 系统消息
      showNotification('系统通知', message.content);
      break;
  }
};

// 后台推送消息
function pushNotification(userId, message) {
  const userWs = connectedUsers.get(userId);
  if (userWs && userWs.readyState === WebSocket.OPEN) {
    userWs.send(JSON.stringify(message));
  }
}
```
